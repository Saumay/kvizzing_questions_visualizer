# KVizzing Visualizer

A SvelteKit web app for browsing, searching, and interacting with the KVizzing trivia question archive.

## Features

- **Question feed** — Date-grouped timeline with full-text search, topic/tag/asker/solver filters, session/non-session toggle, saved questions filter
- **Quiz sessions** — Card gallery with AI-generated background images, question count filters (5+/10+/20+), connect quiz badges
- **Session detail** — Interactive answer submission with multi-part support, connect quiz guess, session save
- **Question detail** — Full discussion thread, media gallery (images/video/audio from R2), like/save/flag actions
- **Highlights** — Topic distribution donut chart, member leaderboards, difficulty breakdown, activity heatmap
- **Review** — Community voting on rejected extraction candidates with reason presets, vote tallies, date-grouped timeline
- **Dark mode** — Full dark/light theme with 5 color schemes
- **Mobile-first** — Responsive layout with mobile pagination, profile menu, and touch-friendly controls

## Setup

```bash
npm install
npm run dev          # Start dev server at http://localhost:5173
npm run build        # Production build (static adapter for Netlify)
npm run preview      # Preview production build
```

## Data

The app reads pre-exported JSON files from `static/data/`:

| File | Description |
|---|---|
| `questions.json` | Full question archive with discussions, media, scores |
| `sessions.json` | Session index with aggregate stats and final scores |
| `stats.json` | Pre-aggregated topic distribution, difficulty, activity |
| `tags.json` | Tag frequency index with question IDs |
| `members.json` | Member profiles with computed stats |
| `rejected_candidates.json` | Rejected extraction candidates for community review |
| `r2_usage.json` | Cloudflare R2 free-tier usage metrics |

These files are generated by the pipeline (`python3 pipeline.py export`).

## Tech Stack

- **SvelteKit 5** with Svelte 5 runes (`$state`, `$derived`, `$effect`)
- **Tailwind CSS 4** for styling
- **Supabase** for persistent user actions (votes, saves, likes, flags)
- **Cloudflare R2** for media file CDN
- **Fuse.js** for client-side fuzzy search
- **Static adapter** for Netlify deployment

## Deployment

Deployed via Netlify with the static adapter. The `netlify.toml` at the repo root configures the build:

```toml
[build]
  base = "v2/visualizer"
  command = "npm run build"
  publish = "v2/visualizer/build"
```
